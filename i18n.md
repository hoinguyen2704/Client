# Kế Hoạch Triển Khai I18n Toàn Dự Án

## Tóm tắt
- Hiện trạng: `src/i18n.ts` đã bootstrap `react-i18next`, nhưng mới chỉ có `2` file dùng `useTranslation`; ngoài `src/locales` vẫn còn khoảng `133` files với `2107` dòng chứa literal tiếng Việt.
- Phạm vi chốt: i18n cho toàn bộ UI hiển thị của frontend, gồm page text, layout, shared constants, toast/error message, formatter ngày giờ; không bao gồm CMS/blog content từ backend, dữ liệu API trả về, route path/slugs, hay comment/JSDoc.
- Rollout chốt: triển khai theo phase, ưu tiên foundation rồi shared/layout/high-traffic public pages trước, sau đó account và admin.
- Ngôn ngữ chốt cho batch này: `vi` và `en`, mặc định `vi`, tiếp tục dùng persisted language trong `ui` store.

## Thay đổi triển khai
- Chuẩn hóa kiến trúc locale theo namespace dưới `src/locales/{lang}/**/*.json`, giữ `settings` và `adminSettings` hiện có, đồng thời bổ sung chính xác các namespace mới: `common`, `layout`, `auth`, `catalog`, `checkout`, `account`, `adminDashboard`, `adminSales`, `adminCatalog`, `adminCustomers`, `adminSupport`, `adminContent`.
- Thay `src/locales/resources.ts` từ import thủ công sang auto-discovery bằng `import.meta.glob(..., { eager: true })`, để thêm namespace mới không phải sửa code bootstrap mỗi lần.
- Giữ `src/i18n.ts` là entry mỏng; `src/locales/config.ts` là nguồn sự thật duy nhất cho `SUPPORTED_LANGUAGES`, namespace list và fallback/default language.
- Quy ước key: JSON lồng nhau theo feature, camelCase semantic keys; không dùng source-string làm key; không để literal tiếng Việt trong component rồi “dịch tạm” sau.
- Thành phần React dùng `useTranslation(namespace)`; chỉ dùng `Trans` cho chuỗi có inline component như link, strong text, hoặc markup.
- Shared constants đang chứa text hiển thị phải đổi từ `label`/`title` sang `labelKey`/`titleKey`. Áp dụng trực tiếp cho menu items, action constants, status constants, modal/confirm constants, order/return/ticket constants.
- Shared helpers không được hardcode text người dùng. `src/utils/error.ts` đổi sang API nhận `t` và `fallbackKey`, ví dụ `getApiErrorMessage(error, t, 'common.errors.unknown')`; mọi message mạng/timeout/common error được đặt trong `common.errors`.
- `src/utils/format.ts` giữ tên hàm hiện tại để không phá call site, nhưng thêm locale-awareness nội bộ. Các hàm `formatDate*` và `formatRelativeTime` đọc ngôn ngữ hiện tại từ i18n nếu không được truyền locale; `formatRelativeTime` dùng `Intl.RelativeTimeFormat` thay cho literal tiếng Việt. `formatPrice` chỉ locale hóa cách hiển thị, không đổi logic currency business theo language trong batch này.
- Bổ sung `common` namespace cho text dùng lặp toàn app: action labels, modal buttons, confirm copy, empty states, generic errors, validation, pagination info, search placeholders, auth common copy, role/status labels chung.
- Bổ sung `layout` namespace cho toàn bộ header/footer/mobile menu/admin sidebar/admin header/user layout/notification dropdown/chat widget chrome; route paths giữ nguyên, chỉ localize label render.
- Bổ sung script audit mới, ví dụ `audit:i18n-literals`, để liệt kê literal tiếng Việt còn lại ngoài `src/locales`; giai đoạn đầu chỉ report, chưa fail CI.
- Rollout bắt buộc theo phase:
  1. Foundation: resource loading, namespace config, helper APIs, audit script, `common`, `layout`, `auth`, formatter, error utils.
  2. Public commerce: `catalog` và `checkout`, ưu tiên `Header`, `Checkout`, `Cart`, `ProductDetail`, `Compare`, `FlashSale`, `Blog`, `Contact`, `Terms`, `Privacy`.
  3. Account: `account` + `settings`, ưu tiên `Profile`, `Orders`, `OrderTracking`, `Vouchers`, `MyReturns`, `ReturnDetail`, `Support`.
  4. Admin: `adminDashboard`, `adminSales`, `adminCatalog`, `adminCustomers`, `adminSupport`, `adminContent`, `adminSettings`, ưu tiên `Dashboard`, `OrderDetail`, `ReturnDetail`, `Vouchers`, `FlashSaleForm`, `Chatbot`, `CMS`, `Tickets`.
- Trong mọi phase, ưu tiên migrate hotspot literals trước các leaf files. Các file đang nặng nhất hiện tại gồm `Checkout`, `Admin Chatbot`, `Settings`, `ReturnDetail`, `CMS`, `Profile`, `FlashSaleForm`, `Admin Vouchers`, `OrderDetail`, `Admin Settings`, `Dashboard`.

## API / Type quan trọng
- `src/locales/config.ts` trở thành contract chính cho `SupportedLanguage`, namespace list và default/fallback; không hardcode `'vi' | 'en'` rải rác nữa.
- `src/utils/format.ts` giữ backward compatibility, nhưng cho phép optional locale/language param và mặc định lấy từ i18n hiện tại.
- `src/utils/error.ts` thay đổi chữ ký để nhận `t` và `fallbackKey`; các call site toast/error phải truyền translator thay vì fallback tiếng Việt cứng.
- Shared constant maps đổi shape từ text cuối cùng sang translation key, ví dụ `{ label: 'Đã giao' }` thành `{ labelKey: 'common.status.delivered' }`.
- Không thêm typed key inference phức tạp ở batch đầu; chỉ type chặt `SupportedLanguage` và namespace, còn key correctness kiểm bằng smoke test + audit.

## Test và tiêu chí chấp nhận
- Verify bootstrap: app khởi động đúng với `language` đã persist trong `ui`; đổi language cập nhật UI mà không cần reload.
- Verify foundation: `Header`, `Footer`, `AdminSidebar`, `AdminHeader`, modal/confirm/action button/status badge đổi ngôn ngữ đúng; không còn literal tiếng Việt trong shared constants đã migrate.
- Verify formatter: ngày giờ và relative time đổi theo `vi`/`en`; currency formatting không phá amount hiện tại.
- Verify page smoke cho cả `vi` và `en`: `Login`, `Register`, `Checkout`, `ProductDetail`, `Profile`, `Orders`, `Dashboard`, `OrderDetail`, `ReturnDetail`, `Tickets`, `CMS`.
- Verify toast/error: network timeout, server unavailable, confirm dialogs, success toasts đều dịch đúng ở cả hai ngôn ngữ.
- Chạy `npm run lint` và `npm run build` sau từng phase; chạy `audit:i18n-literals` để giảm dần literal còn lại. Tiêu chí hoàn tất batch đầu: không còn user-facing literal tiếng Việt trong shared/layout/auth/formatter/error layers.

## Giả định và mặc định
- Không localize backend-provided CMS/article/product data trong plan này; chỉ localize frontend chrome và frontend-owned enum/status labels.
- Không localize route URLs, slug, API field names, hay dữ liệu persisted hiện có.
- Không dịch comment/JSDoc/dev-only strings; audit script chỉ dùng để phát hiện candidate, không coi mọi literal tiếng Việt là bug ngay.
- Monetary currency không đổi theo language switch trong batch này; nếu sau này cần locale + business currency đầy đủ thì làm phase riêng với shop/payment settings.
- Không thêm backend i18n hay SSR; toàn bộ plan là frontend-only trên Vite + `react-i18next`.
